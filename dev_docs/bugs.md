# Bugs

> One entry per bug. Include reproduction, root cause, fix, and
> similar patterns that might harbor the same bug. Cross-reference
> `dev_docs/lessons.md` when the bug taught a general lesson.

## Format

```markdown
## YYYY-MM-DD — <one-line title>  [status: OPEN | FIXED | WONTFIX]

**Reproduction.**
```bash
<commands or steps>
```

**Observed.** What went wrong (error text, exit code, etc.).
**Expected.** What should have happened.
**Root cause.** The actual mechanism.
**Fix.** What changed (commit SHA if landed).
**Similar patterns audited.** Files / functions checked for the
same bug class; result.
**Regression test.** Path + how it would have caught this.
**Related.** Links to `dev_docs/lessons.md` entries if any.
```
